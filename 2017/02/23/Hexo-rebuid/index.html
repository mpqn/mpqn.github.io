重新搭建Hexo博客

发表于 2017-02-23 | 分类于 技术 |

当我们更换电脑或重装系统后，免不了要重新搭建博客，这个时候如果能无缝迁移是最好的，如今遇到了这样一个问题，在网上也查了相关资料，这篇文章是我的笔记，希望对后来者有帮助。

安装环境

  • node.js

  • git

    配置git，并生成新的ssh

    git config –global user.name “your name”
    git config –global user.email “your email”

生成新的ssh公钥，添加到github上

ssh-keygen -t rsa -C "your email"//一路回车即可

安装hexo

npm install hexo-cli -g

将以下文件复制到新的工作目录

config.yml  
theme/  
source/  
scaffolds/  
package.json  
.gitignore

模块安装

npm install

安装插件

npm install hexo-deployer-git --save //hexo d部署到git插件

至此博客已迁移完成，运行以下命令测试是否成功（若有警告忽略即可）。

hexo g   
hexo d

注意

在git bash中切换目录到新拷贝的文件夹里，使用 npm install 命令，进行模块安装。这里没用hexo init初始化，因为有的文件我们已经拷贝生成过来了，所以不必用hexo init去整体初始化，如果不慎在此时用了hexo init，则站点的配置文件_config.yml里面内容会被清空使用默认值，所以这一步一定要慎重。

说明

config.yml  配置  
theme/      主题目录 
source/     博客文件目录

（以上这些肯定要拷贝的） 

scaffolds/   文章模板目录  
package.json 包的使用说明文件
.gitignore   指定忽略提交文件

这三个文件为初始化时自动生成，如果丢失也没关系，可以建立一个新的文件夹，然后在里面执行hexo init，就会生成这三个文件，只需要将它们拷贝过来使用即可。
